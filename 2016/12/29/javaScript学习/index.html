比较运算符

JavaScript在设计时，有两种比较运算符：

第一种是==比较，它会自动转换数据类型再比较，很多时候，会得到非常诡异的结果；

第二种是===比较，它不会自动转换数据类型，如果数据类型不一致，返回false，如果一致，再比较。

由于JavaScript这个设计缺陷，不要使用==比较，始终坚持使用===比较。

null和undefined

null表示一个空的值，而undefined表示值未定义,就是此处应该有一个值，但是还没有定义。
undefined

  • 变量被声明了，但没有赋值时，就等于undefined。
  • 调用函数时，应该提供的参数没有提供，该参数等于undefined。
  • 对象没有赋值的属性，该属性的值为undefined。
  • 函数没有返回值时，默认返回undefined。

闭包

廖雪峰的解释：闭包就是携带状态的函数，并且它的状态可以完全对外隐藏起来。
阮一峰的解释：闭包就是能够读取其他函数内部变量的函数。由于在Javascript语言中，只有函数内部的子函数才能读取局部变量，因此可以把闭包简单理解成”定义在一个函数内部的函数”。所以，在本质上，闭包就是将函数内部和函数外部连接起来的一座桥梁。
综合理解来看，闭包就是能够获得另一个函数作用域中的变量的一个函数。

typeof

typeof操作符获取对象的类型，它总是返回一个字符串。

JavaScript提供了包装对象,number、boolean和string都有包装对象。中，字符串也区分string类型和它的包装类型。包装对象用new创建：

当没有写new，此时，Number()、Boolean和String()被当做普通函数，把任何类型的数据转换为number、boolean和string类型：

  • 不要使用new Number()、new Boolean()、new String()创建包装对象；
  • 用parseInt()或parseFloat()来转换任意类型到number；
  • 用String()来转换任意类型到string，或者直接调用某个对象的toString()方法；
  • 通常不必把任意类型转换为boolean再判断，因为可以直接写if (myVar) {…}；
  • typeof操作符可以判断出number、boolean、string、function和undefined；
  • 判断Array要使用Array.isArray(arr)；
  • 判断null请使用myVar === null；
  • 判断某个全局变量是否存在用typeof window.myVar === ‘undefined’；
  • 函数内部判断某个变量是否存在用typeof myVar === ‘undefined’。
    null和undefined没有toString方法。

正则表达式

  1. 正则表达式图形化界面：www.regexper.com
  2. ECAMscript通过RegExp类型来支持正则表达式。两种方式实例化RegExp：字面量，构造函数。
    • 字面量：var reg=/\bis\b/g;
    • 构造函数：var reg=new RegExp(‘\bis\b’,’g’);

    • i，忽略大小写
    • m，多行搜索
  4. 正则表达式由两种基本字符类型组成：
    • 原义文本字符
    • 元字符（有特殊含义的非字母字符*+？$^.|(){}[]）
  5. 我们可以使用元字符[]来构造一个简单的类（符合某些特征的对象）
    • 字符类
      [abc/g]匹配a或b或c，[^abc/g]取反，匹配a或b或c以外的字符。
    • 范围类
      [0-9A-Z]
    • 预定义类
  6. 量词

    [0-9a-zA-Z_]可以匹配一个数字、字母或者下划线；
    [0-9a-zA-Z_]+可以匹配至少由一个数字、字母或者下划线组成的字符串，比如’a100’，’0Z’，’js2015’等等；
    [a-zA-Z\    \$][0-9a-zA-Z_\$]*可以匹配由字母或下划线、$开头，后接任意个由一个数字、字母或者下划线、$组成的字符串，也就是JavaScript允许的变量名；
    [a-zA-Z_\$][0-9a-zA-Z_\$]{0, 19}更精确地限制了变量的长度是1-20个字符（前面1个字符+后面最多19个字符）。

面向对象编程

JavaScript不区分类和实例的概念，而是通过原型（prototype）来实现面向对象编程。为了解决从原型对象生成实例的问题，Javascript提供了一个构造函数（Constructor）模式。”构造函数”其实是一个普通函数，但是内部使用了this变量。对构造函数使用new运算符，就能生成实例，并且this变量会绑定在实例对象上。

cat1和cat2会自动含有一个constructor属性，指向它们的构造函数。
Javascript规定，每一个构造函数都有一个prototype属性，指向另一个对象。这个对象的所有属性和方法，都会被构造函数的实例继承。
这意味着，我们可以把那些不变的属性和方法，直接定义在prototype对象上。

创建对象

  • 工厂模式
  • 构造函数模式

构造函数本身也是函数。使用大写字母。使用new操作符，实际上经历4个步骤：

1. 创建一个新对象
2. 将构造函数的作用域赋给新对象（this指向新对象）
3. 执行构造函数的代码（为这个新对象添加属性）
4. 返回新对象

新对象有一个constructor（构造函数）属性，指向Person。
如果不使用new，属性和方法都会被添加给window对象。
只要创建了一个新函数，就会为该函数创建一个prototype属性，这个属性指向函数的原型对象。
所有原型对象有一个constructor属性，该属性包含一个指向prototype属性所在函数的指针。
当创建一个新实例，该实例内部含有一个指针（[[Prototype]]）指向构造函数的原型对象。

HTML嵌套规则

  1. 块元素可以包含内联元素或某些块元素，但内联元素却不能包含块元素，只能包含内联元素
  2. 块元素不能放在p里面
  3. 特殊块元素只能包含内联元素，不能再包含块元素，h1-h6,p，dt
  4. li可以包含div标签
  5. 块元素与块元素并列，内联元素与内联元素并列
  6. 块级元素:div h p from ul li dl dt dd;行内元素：span a label strong em input
